# Brighterly Product Prototype

Статический прототип для тестового задания. Главная страница устроена как рабочее поле веб-приложения: сверху переключаются функции, внутри выбранной функции можно переключать приложение/роль и экраны сценария.

Главный сценарий: «Неделя закрепления» для математики 5 класса США. ИИ генерирует задачи по итогам урока, преподаватель проверяет подборку, а ученик проходит три этапа между занятиями:

1. Быстрое закрепление в течение 24 часов после урока.
2. Ежедневная практика на 2–3 задачи с жизнями, зарядом подсказок и пропуском дня.
3. Разогрев за 10 минут до нового урока со шпаргалкой, планом урока и простыми вводными примерами.

## Локальный запуск

Откройте `index.html` в браузере или запустите простой сервер:

```powershell
python -m http.server 8000 --bind 127.0.0.1
```

После этого страница будет доступна на `http://127.0.0.1:8000/`.

## Структура

```text
brighterly-prototype/
  index.html
  css/styles.css
  js/ai-service.js
  js/app.js
```

## ИИ-запросы

ИИ-логика вынесена в `js/ai-service.js`.

Поддержаны два режима:

1. `Демо без API` — безопасный режим по умолчанию, запросы имитируются.
2. `Прямой запрос из браузера` — только для локальной проверки, ключ хранится в `localStorage` или в `js/ai-service.js` и виден в браузере.

Для тестовой демки на GitHub Pages лучше оставить `Демо без API`: интерфейс покажет взаимодействие с ИИ через подготовленные ответы. Если включить прямой ключ в опубликованной версии, он будет виден в браузере.

## Публикация через GitHub Pages

1. Создайте пустой репозиторий на GitHub, например `brighterly-prototype`.
2. Добавьте этот локальный репозиторий как `origin`.
3. Отправьте ветку `main`.
4. В GitHub откройте `Settings` → `Pages`.
5. В разделе `Build and deployment` выберите `Deploy from a branch`.
6. Выберите ветку `main` и папку `/root`.

После публикации адрес будет выглядеть так:

```text
https://YOUR_USERNAME.github.io/brighterly-prototype/
```
